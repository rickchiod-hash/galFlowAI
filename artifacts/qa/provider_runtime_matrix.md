## LLM Provider Selection Matrix

| Provider | Availability Check | Port/Path | Expected Timeout | Quality Levels | Fallback Trigger | Fallback Destination |
|----------|-------------------|-----------|-----------------|---------------|-----------------|---------------------|
| TemplateProvider | `return True` (always) | N/A | Instant | `template`, `fallback` | Never fails | — |
| GPT4AllProvider | gpt4all package importable + model dir exists + .gguf/.bin present | `GPT4ALL_MODEL_DIR` | 120s (generate), 10s (is_available) | `safe`, `fast`, `quality` | generate() returns None/empty/<50 chars | TemplateProvider |
| LMStudioProvider | HTTP GET localhost:1234/v1/models → 200 (2s timeout) | localhost:1234 | 30s (generate), 2s (is_available) | `safe`, `fast`, `quality` | Connection refused/timeout/non-200/generate fails | TemplateProvider |
| KoboldCppProvider | HTTP GET localhost:5001/api/v1/models → 200 (2s timeout) | localhost:5001 | 30s (generate), 2s (is_available) | `safe`, `fast`, `quality` | Connection refused/timeout/non-200/generate fails | TemplateProvider |
| LlamaCppProvider | HTTP GET localhost:8080/v1/models → 200 (2s timeout) | localhost:8080 | 30s (generate), 2s (is_available) | `safe`, `fast`, `quality` | Connection refused/timeout/non-200/generate fails | TemplateProvider |
| Ollama (standalone) | Subprocess `ollama.exe list` exit 0 (10s timeout) | ollama.exe in env path | 60s (generate) | N/A | Binary not found/timeout | Not integrated into router |

## ProviderRouter Mode Behavior

| Mode | Strategy | Order | Fallback |
|------|----------|-------|----------|
| `fast` | Sequential (first available wins) | gpt4all → llamacpp → koboldcpp → lmstudio → TemplateProvider | TemplateProvider if all fail |
| `quality` | Parallel (first valid result wins) | gpt4all + llamacpp + koboldcpp + lmstudio (concurrent) → TemplateProvider | TemplateProvider if all fail |
| `safe` | Sequential sync | gpt4all → llamacpp → koboldcpp → lmstudio → TemplateProvider | TemplateProvider if all fail |

## Explicit Provider Selection (UI/API)

| Provider Selected | Action | Success Path | Fallback Path |
|------------------|--------|-------------|---------------|
| `template` | Direct call to TemplateProvider | Script generated from template | — |
| `lmstudio` / `koboldcpp` / `llamacpp` / `gpt4all` | Generate template context → build enhanced prompt → call specific provider with 120s timeout | Script generated by real LLM | TemplateProvider if fail/timeout/empty |

## Video Rendering Fallback Chain

| Step | Provider | Success Condition | Fallback | Fallback Condition |
|------|----------|-----------------|----------|-------------------|
| Per-scene render | WanGPAdapter | Scene MP4 generated | FFmpegAdapter (static text video) | WanGP unavailable or error |
| Per-scene fallback | FFmpegAdapter | Static MP4 generated | Scene marked "failed" | FFmpeg not found or error |
| Final concat | FFmpegAdapter | All completed scenes concatenated | Error "No scene videos generated" | Zero completed scenes |

## Provider Status Endpoint Behavior

| Scenario | `/api/v1/llm/providers` Response |
|----------|----------------------------------|
| All offline (no LLM servers running) | template: true, others: false |
| GPT4All available | gpt4all: true (if package+model present) |
| LM Studio running | lmstudio: true (if :1234 responds) |
| KoboldCpp running | koboldcpp: true (if :5001 responds) |
| LlamaCpp running | llamacpp: true (if :8080 responds) |
